轻量应用服务器配置crontab后无效

May 25, 2023 · 1 min · 冲和
Table of Contents
  • 背景
  • 问题
  • 解决

背景#

需要定时同步订单数据，所以在阿里云轻量应用服务器上配置了 crontab， 发现并未生效，但是手动执行命令是可以的。

问题#

crontab -e 配置定时任务后，未生效。

解决#

怀疑是环境变量未获取到的原因， 就在定时任务前加了 source /etc/profile，至此解决。

# 之前的配置
* * * * * cd xxx && php artisan schedule:run >> /dev/null 2>&1

# 修改之后的配置
* * * * * source /etc/profile;cd xxx && php artisan schedule:run >> /dev/null 2>&1

具体可参考这篇文章 【转载】Linux 中 Crontab 执行时的环境变量问题
