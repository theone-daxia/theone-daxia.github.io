laravel 设置 https 的几种情况#

提交表单，提示不安全问题#

<!-- 原来的写法 -->
<form action="{{ route('order/import') }}" method="POST" enctype="multipart/form-data"></form>

<!-- 修改后的写法 -->
<!--  -->
<form action="{{ secure_url('order/import') }}" method="POST" enctype="multipart/form-data"></form>

即使用 secure_url() 方法替换 route() 方法即可。

注意：使用 secure_url() 时，如果不是相对路径，则需要将第三个参数设置为 false， 即 secure_url(‘your path’, [], false)。

全局使用 https#

在使用相关框架做管理后台开发，比如 nova、laravel-admin、orchid 等， 需要配置使用 https 时，有以下两种方法。

1. 修改 nginx 配置#

在对应项目的 nginx 配置中，添加 fastcgi_patam HTTPS on;

location ~ [^/]\.php(/|$) {
  #fastcgi_pass remote_php_ip:9000;
  fastcgi_pass unix:/dev/shm/php-cgi.sock;
  fastcgi_index index.php;
  fastcgi_param HTTPS on;
  fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
  include fastcgi_params;
}

2. 修改项目代码#

在项目代码 app/Providers/AppServiceProvider.php 文件的 boot() 方法中，添加如下代码

\Illuminate\Support\Facades\URL::forceScheme('https');

但是这种修改方式还是可能出现生成链接错误的情况， 比如 route(config(platform.index))， 所以如果你的项目全局都用 https，直接修改对应 nginx 配置最好。

总结#

其实 laravel 不关心 url 是否安全，它是基于当前 url 生成的。 如果在 https 页面上，route() 将返回安全的网址， 如果在 http 页面上，将返回 http:// 网址。

由于使用了 nginx，监听 443 端口，将请求转发到 8081端口，laravel 接收到的 request 都会是 http。

所以如果需要全局使用 https，最好是修改对应项目的 nginx 配置。
