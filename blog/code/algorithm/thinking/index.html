数组技巧#

前缀和#

  1. 适用场景

    适用于原始数组不会被修改的情况下，频繁的查询某个区间的累加和。

  2. 构造方式

    nums := []int{1,2,3}
preSum := make([]int, len(nums)+1)
for i := 1; i < len(nums); i++ {
    preSum[i] = preSum[i-1] + nums[i-1]
}

  3. 题目推荐（leetcode）

    • 303. 区域和检索-数组不可变
    • 304. 二维区域和检索-矩阵不可变

差分数组#

  1. 适用场景

    适用于频繁对原始数组某个区间的元素进行增减。

  2. 构造方式

    nums := []int{1,2,3,4,5}
diff := make([]int, len(nums))
diff[0] = nums[0]
for i := 1; i < len(nums); i++ {
    diff[i] = nums[i] - nums[i-1]
}

// 如果要对原始数组下标 1-3 区间的元素都加 2，只需做如下两步操作
diff[1] += 2
diff[4] -= 2

// 最后根据 diff 来构造更新后原始数组
res := make([]int, len(diff))
res[0] = diff[0]
for i := 1; i < len(diff); i++ {
    res[i] = res[i-1] + diff[i]
}

  3. 题目推荐（leetcode）

    • 1094. 拼车
    • 1109. 航班预订统计

滑动窗口#

  1. 适用场景

    适用于能回答出以下 3 点问题的子串、子数组问题：

    1. 什么时候扩大窗口？
    2. 什么时候缩小窗口？
    3. 什么时候更新结果？

  2. 代码框架

    left, right := 0, 0
for right < len(s) {
    in := s[right]
    right++ // 扩大窗口
    ... // 窗口数据更新

    for {
        out := s[left]
        left++ // 缩小窗口
        ... // 窗口数据更新
    }
}

动态规划#

  1. 适用场景

    求最值问题，且具备最优子结构，即能通过子问题的最值得到原问题的最值。

    三要素：最优子结构、重叠子问题、状态转移方程。

  2. 框架

    • 明确 base case
    • 明确状态（原问题和子问题中会变化的变量）
    • 明确选择（导致状态产生变化的行为）
    • 定义dp数组/函数的含义
    1. 自顶向下：会存在一个递归的 dp 函数
    2. 自底向上：会存在一个 dp 数组

  3. 题目推荐（leetcode）

    322. 零钱兑换 509. 斐波那契数

回溯算法#

  1. 适用场景

    排列、组合、子集问题。

    思路：解决一个回溯问题，实际上就是一个决策树的遍历过程，站在决策树的一个节点上，你只需要思考 3 个问题：

    1. 路径：已作出的选择
    2. 选择列表：当前可以做的选择
    3. 结束条件：到达决策树的底层，无法再做选择的条件

  2. 框架

    res := [][]int{}
backtrack(路径, 选择列表) {
    if 满足结束条件 {
        res = append(res, 路径)
    }

    for 选择列表 {
        做选择
        backtrack(路径, 选择列表)
        撤销选择
    }
}

    • 51. N 皇后
    • 77. 组合
    • 78. 子集
